Вариант 1. Сложение длинных чисел
	Требуется написать программу с полным контролем ошибок. С клавиатуры вводится основание системы счисления base_input для исходных чисел, где 2<=base_input<=36 (в случае, если основание base>10, то для ввода цифр используются буквы латинского алфавита); основание системы счисления base_output; два числа num_left, num_right длины не более 1000 цифр. Оба числа принадлежат множеству целых чисел (отрицательное число тоже может встретиться).
	Программа должна вывести значение суммы чисел num_left, num_right в системе счисления base_output.
___________________________________

Из-за ограничений по памяти и времени нельзя складывать длинные числа так же, как и обычные. Поэтому в моей программе реализован следующий алгоритм сложения длинных чисел:

Используются методы динамического программирования. Каждый цифра числа считывается отдельно как символ (в цикле), проверяется и после попадает в соотвествующий массив (num_left, num_right);

Само сложение происходит по принципу "сложения в столбик", то есть мы поразрядно складываем цифры. При необходимости "запоминаем" и прибавляем единицу в более старший разряд. (Обращение к разрядам реализовано через цикл)

В программе рассматриваются разные ситуации: 
num_left отрицательное, num_right положительное
num_left положительное, num_right отрицательное
num_left и num_right одного знака: оба положительные или оба отрицательные

Помимо этого числа сравниваются по длине.
Служебная переменная length_comparison выполняет роль флага. 
Она принимает значения: 0 если числа равны по длине, 1 если num_left длиннее, -1 если num_right длиннее

----------------------------------
Пользовательский интерфейс

Для удобства заголовки сделаны бирюзовым цветом, ответы пользователя - белым, ошибки - красным, верный ответ - зелёным. Такое оформление делает использование программы более удобным и интуитивно понятным.

В случае ошибок пользователю предлагается продолжить (попробовать ввести заново) или прекратить выполнение программы.

---------------------------------
Формат входных данных и ошибки.

Системы счисления base_input и base_output имеют ограничения (они должны быть не меньше 2 и не больше 36). В случае если введено число, не соответствующее ограничениям, то будет выведена ошибка, и будет предложено ввести новое число до тех пор, пока оно не будет удовлетворять условию.

Числа num_left и num_right следует вводить через пробел.
В случае если числа были введены с ошибкой (длиннее 1000 символов / в их записи были посторонние символы / не через пробел / одно из чисел или оба - пустая строка и т.д.), то программа выведет ошибку и завершится.
